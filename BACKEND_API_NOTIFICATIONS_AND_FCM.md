# Mobile Push Notifications & FCM Integration Guide

This guide is for the mobile (Flutter) developer implementing real-time Firebase Cloud Messaging (FCM) and the notifications inbox.

---

## 1. Firebase Cloud Messaging (FCM) Integration

The backend is integrated with the `firebase-admin` SDK. When a chat message or a subscription/billing update occurs, a multicast push notification is sent to all registered FCM tokens for the target user.

### Device Token Registration

Register the device token whenever the user logs in or a new token is generated by FCM.

* **Endpoint:** `POST /v1/notifications/fcm-token`
* **Headers:** `Authorization: Bearer <USER_JWT_TOKEN>`
* **Request Payload:**
  ```json
  {
    "token": "d82jdK9sL0...YOUR_FCM_REGISTRATION_TOKEN...",
    "platform": "android", // "android" or "ios"
    "deviceId": "unique_device_id_here" // Optional, helps prevent duplicate rows for the same device
  }
  ```
* **Response Status:** `201 Created`

### Device Token Unregistration

Unregister the device token when the user logs out to prevent them from receiving pushes on that device.

* **Endpoint:** `DELETE /v1/notifications/fcm-token`
* **Headers:** `Authorization: Bearer <USER_JWT_TOKEN>`
* **Request Payload:**
  ```json
  {
    "token": "d82jdK9sL0...YOUR_FCM_REGISTRATION_TOKEN..."
  }
  ```
* **Response Status:** `200 OK`

## 2. Push Notification Payload Formats

Notifications sent via FCM contain two sections:
1. **`notification`**: Contains `title` and `body` strings which the OS uses to render the system tray banner automatically.
2. **`data`**: Key-value pairs of custom payloads used to navigate the user inside the app or update local state.

### A. Chat Messages Payload (Agent / Admin chat)
Triggered when the agent or admin receives a new message.

```json
{
  "notification": {
    "title": "New Message from Admin", // Or "New Message from <Agent Name>"
    "body": "Hi, please review the latest customer leads."
  },
  "data": {
    "id": "142",                     // Database Notification ID
    "type": "chat",                  // Notification category
    "companyId": "5",                // Target company/agent account ID
    "messageId": "1002",             // ID of the new message
    "senderId": "1"                  // User ID of the sender
  }
}
```

### B. Billing & Subscription Updates
Triggered when an admin updates the agent's subscription/billing settings or when the trial/subscription is ending within 3 days (JIT).

```json
{
  "notification": {
    "title": "Subscription Plan Updated", // Or "Trial Period Expiring Soon" / "Subscription Expiring Soon"
    "body": "Your subscription plan has been updated to \"Pro Plan\"."
  },
  "data": {
    "id": "143",
    "type": "billing",
    "companyId": "5",
    "updateType": "subscription"     // e.g. "subscription"
  }
}
```

## 3. In-App Notifications Inbox API

To build the notifications inbox list/drawer, use the following endpoints:

### Retrieve Notifications List
Fetch the paginated notification log for the authenticated user.

* **Endpoint:** `GET /v1/notifications?filter=all&page=1&pageSize=20`
* **Headers:** `Authorization: Bearer <USER_JWT_TOKEN>`
* **Query Parameters:**
  - `filter`: Filter notifications. Supported values: `all` (default), `unread`, `read`, `archived`
  - `page`: Page index (default: `1`)
  - `pageSize`: Number of items per page (default: `20`, max: `100`)
* **Response Payload:**
  ```json
  {
    "data": [
      {
        "id": 142,
        "userId": 10,
        "title": "New Message from Admin",
        "body": "Hi, please review the latest customer leads.",
        "type": "chat",
        "isRead": false,
        "isArchived": false,
        "contactId": null,
        "senderAvatarUrl": "http://127.0.0.1:8080/uploads/avatars/admin_profile.png", // Absolute URL, nullable
        "createdAt": "2026-06-06T09:20:00.000Z",
        "updatedAt": "2026-06-06T09:20:00.000Z"
      }
    ],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "totalItems": 1,
      "totalPages": 1
    },
    "unreadCount": 1
  }
  ```

> [!TIP]
> **Sender Avatar Display**: Use `senderAvatarUrl` to render the sender's avatar inside the notification cell. If it is `null` or fails to load, fall back to the first letter of the notification `title` or a standard system billing icon if `type` is `"billing"`.

### Mark Notification as Read
Update the status of a specific notification to read.

* **Endpoint:** `PATCH /v1/notifications/:id/read`
* **Headers:** `Authorization: Bearer <USER_JWT_TOKEN>`
* **Response Payload:**
  ```json
  {
    "notification": {
      "id": 142,
      "isRead": true
    }
  }
  ```

### Archive Notification
Remove the notification from the standard inbox filter.

* **Endpoint:** `PATCH /v1/notifications/:id/archive`
* **Headers:** `Authorization: Bearer <USER_JWT_TOKEN>`
* **Response Payload:**
  ```json
  {
    "notification": {
      "id": 142,
      "isArchived": true
    }
  }
  ```

### Mark All Notifications as Read

* **Endpoint:** `POST /v1/notifications/read-all`
* **Headers:** `Authorization: Bearer <USER_JWT_TOKEN>`
* **Response Payload:**
  ```json
  {
    "markedCount": 3
  }
  ```
